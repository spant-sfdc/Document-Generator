0 Introduction ~2 min

TALK

"Every nonprofit has the same challenge: your donors give generously, and you need to acknowledge that in a timely, professional, and personalised way — while your team is already stretched thin. Today I'm going to show you how DocGen, built natively in Salesforce, lets you go from a live donation record to a fully-merged, branded PDF receipt in under two minutes — without any code, any middleware, or any third-party licence."

TALK

"We're going to follow a real Opportunity — the Education Support Fund campaign — from raw data all the way through to a printed acknowledgement letter. Along the way I'll show you three things that usually require custom development: pulling in parent account data, repeating line items from child records, and conditional messaging for major donors."

1 Upload the HTML Template ~2 min

What to do

  1. CLICK Navigate to the Education Support Fund – FY26 Opportunity record.
  2. POINT The DocGen Wizard component in the right-hand sidebar (or embedded tab).
  3. CLICK New Document Template.
  4. CLICK Upload HTML Template and select donation-receipt-template.html from your desktop.
  5. POINT The token extraction progress bar — the wizard scans the file and extracts every {{token}} automatically.
TALK

"The template is just a plain HTML file your team designs in any editor. It uses double-curly-brace placeholders — like a mail-merge field. The wizard reads the file and lists every placeholder it finds. No schema mapping required yet — that comes in the next step."

POINT Highlight these tokens when the extraction list appears:
  • {{donor_name}} — parent Account field (cross-object)
  • {{#line_items}} — repeating child-record section opener
  • {{#if isMajorDonor}} — conditional block for major donors
  • {{sys.today}} — system variable (no mapping needed)
TALK

"Notice the system tokens like sys.today and sys.orgName — those resolve automatically from org configuration. The team never has to hard-code a date or update org details in the template."

2 Select Objects ~1 min

What to do

  1. CLICK Next: Select Objects.
  2. POINT Primary Object = Opportunity (auto-selected from the record context).
  3. CLICK Add Parent Object: Account.
  4. CLICK Add Child Object: OpportunityLineItem.
TALK

"We're telling the wizard which objects contain our data. The primary is the Opportunity we're generating from. The parent — Account — is the donor organisation whose name and ID we want on the receipt. The child — OpportunityLineItem — is the donation line-item breakdown that will populate our repeating table."

3 Map Tokens — Auto Fill Demo ~2 min

What to do

  1. CLICK Next: Map Tokens.
  2. POINT The blue demo banner at the top of the mapping table: "Demo: Donation Receipt — Education Support Fund".
  3. CLICK Auto Fill Mapping.
  4. POINT Watch all token rows populate instantly.
TALK

"In a live engagement you'd map each field yourself — which takes about a minute. For this demo I'll use the auto-fill, which applies a pre-configured scenario in one click. Notice it handles three different mapping patterns simultaneously."

Highlight these three patterns after auto-fill

What to point atWhat to say
{{total_donation}} row — Format: currency "Primary object field, automatically formatted as a dollar amount."
{{donor_name}} row — Source: Account, Relationship Path: Account.Name "This is a cross-object lookup. The wizard traverses the standard Account relationship from the Opportunity — no SOQL, no formula fields, no Apex needed."
{{#line_items}} row — Repeating: Yes, Child: OpportunityLineItem "This is the section marker. Everything between the opening and closing tag will repeat once per donation line item. Child fields like quantity, unit price, and line amount are each mapped to their OpportunityLineItem fields."
Recovery tip: If the auto-fill button does not appear, confirm that the Demo_Mode__mdt.Donation_Demo record has Is_Enabled__c = true and re-open the wizard.

4 Configure Variables (Conditional Section) ~2 min

What to do

  1. CLICK Next: Variables.
  2. POINT The pre-detected variable isMajorDonor — extracted automatically from the {{#if isMajorDonor}} block in the template.
  3. CLICK Auto Fill Variables.
  4. POINT The expression fills in: total_donation > 10000 with type Boolean.
TALK

"Variables power the conditional logic. This expression — total_donation greater than 10,000 — evaluates at generation time against the live Opportunity Amount. Because this donation is $15,000, the condition will be true, and the major donor banner will appear in the document. Change the threshold, change the message — no code, no Apex trigger, no workflow rule."

POINT In the template preview pane (if visible), highlight the conditional block: 
{{#if isMajorDonor}}
  <div class="banner">Thank you for your major gift!</div>
{{/if}}

"This entire block is invisible for any donation under $10,000. It appears automatically for major gifts."

5 Preview & Download ~3 min

What to do

  1. CLICK Next: Preview.
  2. POINT The live HTML preview rendered with actual record data.
  3. Walk through each section of the preview (see talking points below).
  4. CLICK Download PDF.
  5. POINT The generated file appears in the Files related list on the Opportunity record automatically.

Preview walkthrough talking points

Section in previewWhat to say
Organisation header (org name, EIN, address) "These come from DocGen Config — a single place to maintain your org's letterhead details across every document type."
Donor name & ID "{{donor_name}} and {{donor_id}} pulled from the related Account — no relationship fields or formulas on the Opportunity record needed."
Close date formatted as May 1, 2026 "Raw Salesforce date fields often come through as ISO strings. A one-field format override — date:MMMM d, yyyy — makes it donor-friendly automatically."
Donation line items table (repeating rows) "Each row here is a live OpportunityLineItem from the record. Two line items in Salesforce, two rows in the PDF. Add a third line item tomorrow, regenerate, and you'll see three rows — zero template changes."
Major donor banner (conditional block) "This banner shows because $15,000 satisfies our Boolean expression. It won't appear on the next donation at $5,000 — same template, context-aware output."
Files related list on the record "The PDF is automatically attached to the Opportunity as a Salesforce File. Your team's donor history is complete without anyone manually uploading anything."
TALK

"What you just saw — cross-object data, repeating child records, and conditional sections — would typically require a custom Apex class, a Visualforce template, and a developer sprint. With DocGen, an admin configured the entire thing in five steps, in under five minutes, without writing a single line of code."

Discovery Questions

Use these after the demo to move the conversation forward:

  • "What types of documents does your team currently send to donors — receipts, grant agreements, pledge reminders?"
  • "How are those documents generated today — Word mail merge, a third-party tool, manual editing?"
  • "Who on your team owns the templates? Is that a developer task or something an admin or program officer can handle?"
  • "Do you need different document formats for different donor segments — major donors, recurring donors, corporate sponsors?"
  • "How important is automatic file attachment to the donor or grant record for your audit trail?"

Common Objections & Responses

ObjectionResponse
"We already use [third-party tool]." "DocGen is native — no API calls, no data leaving Salesforce, no per-document licence fee. And because it's inside Salesforce, every generated document is automatically a Salesforce File with the full audit trail. Does your current tool attach the PDF directly to the record?"
"Our templates are complex — tables within tables, signatures, logos." "The template is plain HTML, so anything you can design in HTML you can include — logos via image tags, nested tables, branded CSS. The token syntax sits on top of whatever HTML you already have."
"We need Word output, not PDF." "DocGen generates both — the Download Word button is right next to Download PDF on the preview screen."
"What about bulk generation — we send 500 receipts at year end." "Bulk generation is on the roadmap. Today the wizard is record-by-record, which covers on-demand receipts and letters. For the year-end batch run, we'd discuss the timeline and whether a triggered or scheduled flow approach fits your org's setup."

Demo Reset Instructions

To reset the environment for the next audience:

  1. Delete the generated PDF from the Opportunity's Files related list.
  2. Delete the Document Template record created during the demo (navigate to Document Templates tab → delete).
  3. The Opportunity record itself does not need to change — the same record can be reused.
Note: Do not disable Demo_Mode__mdt.Donation_Demo between runs — the auto-fill buttons rely on it being enabled.
