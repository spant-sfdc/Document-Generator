DocGen – Document Generation Module: User Guide

Version: 1.0  |  Last Updated: May 2026  |  Audience: End Users & Salesforce Admins

Overview

DocGen is a native Salesforce document generation tool that merges live record data into HTML or PDF documents from reusable templates. It supports dynamic field mapping, conditional sections, repeating child-record tables, and custom variables — all configured through a guided wizard without writing code.

Typical use cases include donation receipts, grant agreements, client summary letters, and any document that combines data from multiple related Salesforce objects.

Key Features

  • Upload HTML templates with {{token}} placeholders
  • Map tokens to any field on the primary, parent, or child object
  • Conditional sections using {{#if variableName}} blocks
  • Repeating child-record tables using {{#sectionName}} blocks
  • System variables (today's date, org name, EIN) resolved automatically
  • Format overrides for dates, currencies, and numbers
  • Preview in-browser before generating the final PDF or Word document
  • Demo Mode autofill for quick setup in sandbox/demo environments

Step-by-Step Usage

1 Upload Template

Navigate to a record page that has the DocGen Wizard component. Click New Document Template and upload your HTML file.

  • The template must be a valid .html file.
  • Tokens must use double-curly-brace syntax: {{field_name}}
  • After upload, the wizard extracts all tokens automatically and advances to the next step.
Tip: Use the provided donation-receipt-template.html as a starting reference. It includes all supported token types.

2 Select Objects

Choose the Primary Object (the record type you are generating the document from), any Parent Objects (lookup targets), and any Child Objects (related lists).

  • Primary Object – e.g. Opportunity
  • Parent Objects – e.g. Account (the donor)
  • Child Objects – e.g. OpportunityLineItem (donation line items)

3 Map Tokens

Every {{token}} found in the template appears as a row in the mapping table. For each token, configure:

ColumnDescription
TokenThe placeholder exactly as it appears in the template, e.g. {{donation_name}}
TypeField (from a record), Constant (static text), or Variable (computed)
Source ObjectWhich object the field lives on (Primary, Parent, or Child)
Source FieldThe API field name, e.g. Amount, Donor_ID__c
FormatOptional override: currency, date:MMMM d, yyyy, upper, etc.

If Demo Mode is enabled, click Auto Fill Mapping to populate all rows from the pre-configured scenario instantly.

4 Variables (IF Conditions)

Variables are used in {{#if variableName}} conditional blocks. Auto-detected variable names (from your template) appear pre-filled.

TypeBehaviourExample Expression
BooleanEvaluates a comparison against a record field; shows the conditional block when truetotal_donation > 10000
TextInjects a static string valueMajor Donor
NumberInjects a static numeric value5000
DateInjects a static date string2026-12-31
Boolean expressions support the operators >, <, >=, <=, ==, !=, and null. The left-hand side is a token name (e.g. total_donation) which resolves to the mapped field value.

5 Preview & Download

Click Next: Preview to generate a live HTML preview populated with actual record data. Review the output, then click Download PDF or Download Word to produce the final file. The generated document is automatically attached to the record as a Salesforce File.

Token Reference

Field Tokens

Any field on the primary or related objects can be referenced by its token name:

  • {{donation_name}} – mapped to Opportunity.Name
  • {{total_donation}} – mapped to Opportunity.Amount
  • {{donor_name}} – mapped to Account.Name via parent relationship

System Tokens (auto-resolved, no mapping required)

TokenResolves To
{{sys.today}}Today's date
{{sys.currentUser}}Running user's full name
{{sys.orgName}}Organisation display name (from DocGen Config)
{{sys.orgEIN}}Federal EIN (from DocGen Config)
{{sys.orgAddress}}Organisation address (from DocGen Config)
{{sys.orgPhone}}Organisation phone (from DocGen Config)
{{sys.orgWebsite}}Organisation website (from DocGen Config)

Conditional Blocks

Wrap any HTML in a conditional block. The block renders only when the named variable evaluates to true.

{{#if isMajorDonor}}
  <div class="banner">Thank you for your major gift!</div>
{{/if}}

Repeating Child-Record Tables

Use a section block to repeat a row template for every child record. The section name must match the child object relationship token configured in the mapping step.

{{#line_items}}
<tr>
  <td>{{contribution_type}}</td>
  <td>{{quantity}}</td>
  <td>{{unit_price}}</td>
  <td>{{line_amount}}</td>
</tr>
{{/line_items}}

Common Errors & Fixes

SymptomCauseFix
Token appears literally in output (e.g. {{donor_name}}) Token has no mapping or mapping type is set to Variable without a defined variable Open the Mapping step and confirm the token has a Source Object and Source Field selected
Child table rows are empty Child field mappings do not have the correct Source Object or the Repeat Object is blank Ensure each field token inside a section block maps to the child object, not the primary object
Conditional block never shows Variable expression references a token name that is not mapped, so the value is always null Confirm the left-hand side of the Boolean expression matches the token name used in a field mapping (e.g. total_donation, not Amount)
Parent field (e.g. donor name) is blank Relationship path is not set; the system tried to query the field directly on the primary object Ensure the mapping for a parent field has a Relationship Path set (e.g. Account.Name)
Date shows raw ISO format instead of formatted text No Format Override set for a date field Set Format Override to date:MMMM d, yyyy (or any Java SimpleDateFormat pattern)
Currency shows without dollar sign or decimal No Format Override set for a currency field Set Format Override to currency
"Invalid field" error during generation FLS — the running user does not have read access to the mapped field Assign the DocGen Full Access or DocGen Demo Fields permission set to the user
